  • Afficher les services occupés dans l'entreprise : SELECT service FROM employes;

Les condition : le mot-clef WHERE

Nous cherchons avec une condition une partie des données

Affiche moi le nom et le prénom de la table des employés à condition que les employés fassent parti du service informatique

  • SELECT nom, prenom FROM employes WHERE service='informatique';
  • Ou l'employé dont le prénom est thomas : SELECT NOM, prenom FROM employes WHERE prenom='thomas';

Le mot-clef BETWEEN

Afficher les employés ayant recrutés entre 2006 et 2010

  • SELECT * FROM employes WHERE date_embauche BETWEEN '2006-01-01' AND '2010-12-31';
  • SELECT nom, prenom, date_embauche FROM employes WHERE date_embauche BETWEEN '2006-01-01' AND '2010-12-31';
  • Dans une BDD la date est au format états-unien ANNEE-MOIS-JOUR // 2021-02-23
  • Une requête qui donne la date du serveur SELECT CURDATE();
  • Entre une date et aujourd'huiSELECT * FROM employes WHERE date_embauche BETWEEN '2006-01-01' AND CURDATE();

Le mot-clef LIKE

Afficher les employés dont le prénom commence par s, ce qui sera très utile dans un moteur de recherche

Avec la présence des signes pourcentage avant et après la lettre ou le mot recherché, nous demandons au système de trouver des enrengistrement contenant l'expression recherchée.

  • SELECT * FROM employes WHERE prenom LIKE 's%';
  • SELECT * FROM employes WHERE prenom LIKE '%-%';

7- Limiter les resultats avec le mot-clef LIMIT

  • Afficher les employés en les classant par salaire (du + grand au plus petit) et en les affichant par 3 SELECT nom, prenom, service, salaire FROM employes ORDER BY salaire DESC LIMIT 3;
  • le premier chiffre précise l'enrengistrement de départ (0 définit le 1er enrengistrement en informatique), le 2nd chiffre nous indique le nombre de résultats que l'on souhaite obtenir
  • AUTRE ex. : SELECT nom, prenom, service, salaire FROM employes ORDER BY salaire DESC LIMIT 6,3;
  • le mot-clef ou la close LIMIT est utile pour faire une pagination

8- Réaliser des calculs dans nos requêtes

  • Afficher la liste des employés avec leur salaire annuel : SELECT nom, prenom, salaire, salaire*12 FROM employes ORDER BY salaire;

Utiliser un alias avec AS avec le mot-clef AS

  • SELECT nom, prenom, salaire, salaire*12 AS salaire_annuel FROM employes ORDER BY salaire_annuel DESC;
  • Le mot-clef AS permet de définir un alias

Calculer une somme avec SUM, et une moyenne avec AVG

  • SUM faire une somme, une addition : SELECT SUM(salaire*12) FROM employes;
  • AVG faire la moyenne (average)SELECT AVG(salaire*12) FROM employes; ou la moyenne mensuelle : SELECT AVG(salaire) FROM employes;

Arrondir avec ROUND

  • Arrondir un calcul : SELECT ROUND( AVG(salaire) ) FROM employes;
  • Faire un alias SELECT ROUND (AVG(salaire)) AS moyenne_mensuelle from employes;

Compter avec COUNT

  • Compter le nombre d'employés : SELECT COUNT(*) FROM employes WHERE sexe='f';

Isoler une valeur minimum/maximum avec MIN ou MAX

  • Trouver le salaire minimum : SELECT MIN(salaire) FROM employes;
  • Par contre cette requête ne fonctionnera pas : SELECT prenom, MIN(salaire) FROM employes;, pour trouver le salaire le plus bas et la personne qui gange ce salaire il faut écrire SELECT prenom, salaire FROM employes WHERE salaire = (SELECT MIN(salaire) FROM employes);: Le système va d'abord exécuter la requête entre parenthèse pour conaître le resultat et utiliser comme source pour l'autre requête
  • Trouver le salaire maximum :SELECT MAX(salaire) FROM employes;
  • Trouver qui gangne le plus : SELECT prenom, salaire FROM employes WHERE salaire = (SELECT MAX(salaire) FROM employes);
  • Trouver qui gagne 1390 Euros : SELECT nom, prenom, salaire FROM employes WHERE salaire = 1390;

Condition IN avec plusieur valeurs

  • Trouver les employés à la comptabilité et au service informatique SELECT nom, prenom, service FROM employes WHERE service IN('comptabilite', 'informatique'); on appelle cela une requête imbriquée.
  • Trouver le salaire à l'heure des personneSELECT nom, prenom, salaire, salaire/150 AS salaire_heure FROM employes ORDER BY salaire_heure ASC;
  • ...
  • = (égal) permet d'annoncer 1 seul valeur
  • IN permet d'annoncer plusieurs valeurs
  • ...
  • Trouver les employés qui ne sont pas ua service informatique et à la comptabilité avec NOT INSELECT nom, prenom, service FROM employes WHERE service NOT IN ('comptabilité', 'informatique');

COndition multiples, différents champs et valeurs

  • Les employés qui travaillent au service commercial et qui gagne un salaire inférieur ou égal à 2000 €SELECT nom, prenom, salaire, service FROM employes WHERE service = 'commercial' AND salaire <= 2000;

Ordre de priorité sur les conditions AND et OR

  • SELECT nom, prenom, service, salaire FROM employes WHERE service ='production' AND salaire = 1900 OR salaire = 2300;
  • Lorsqu'il y a AND et OR au sein de la même requête, le système lit la condition de la droite vers la gauche et il verra d'abord salaire = 1900 OR salaire = 2300;et ensuite service ='production';
  • Pour garantir l'odre naturel de lecture (dela gauche vers la droite) on utilisera les parenthèses : SELECT prenom, nom, service, salaire FROM employes WHERE service='preduction' AND (salaire= 1900 OR salaire= 2300);

Les regroupements avec groupe BY

  • Si on veut connaître le nombre d'employés par service : SELECT service, COUNT(*) AS nombre_employe FROM employes GROUP BY service;

Utilisation du mat-clef HAVING, ayant

  • On impose une condition COUNT et HAVINGSELECT service, COUNT(*) AS nombre_employe FROM employes GROUP BY service HAVING COUNT(*) > 2;

Les requêtes de modification avec UPDATE

La requête UPDATE

  • Augmenter le salaire d'un employé : UPDATE employes SET salaire = 1871 WHERE id_employes = 990;
  • Il est recommandé de modifier une enregistrement en ciblant son id, l'id étant unique
  • ...
  • Pour modifier plusieur colonnes respectons l'ordre des colonnes : UPDATE employes SET service = 'commercial', salaire = 1999 WHERE id_employes = 990;

La requête DELETE

  • Supprimer un enrengistrement ; DELETE FROM employes WHERE id_employes = 388;
  • Supprimer plusieurs informations, plusieurs enrengistrement : DELETE FROM employes WHERE service = 'informatique';
